# priceTag_parser

MVP-проект Android-приложения для распознавания магазинных ценников (RU) и сохранения данных в таблицу.

## Цель MVP

Сканировать **один ценник в кадре** и извлекать поля:
- название товара;
- цена;
- цена за кг;
- вес/объём.

Приоритет: **точность выше скорости**. Всё — на бесплатном/open-source стеке.

---

## Подтверждённые продуктовые решения (зафиксировано)

- Режим использования: **личный**.
- Платные облака: **не используем**.
- Старт: 1–2 формата ценников, далее расширяем.
- В кадре: **ровно 1 ценник**.
- Android target: ориентир **Android 10+**, основной тест — Android 13 (Redmi Note 10S).
- Язык ценников: в основном **русский**.
- Live-режим: нужен, но сохранение только **после подтверждения**.
- Сессии: каждый вход в live-режим = **новая сессия сканирования**.
- Магазин/адрес: пока **не храним**.

### Нормализация полей
- Вес/объём: поддерживаем смешанные форматы (`г`, `кг`, `мл`, `л`).
- Цена за кг: маркер бывает не всегда, поэтому парсим и по контексту/позиции.
- Копейки: допускаем целые значения; при сомнительной дробной части применяем округление вверх (по правилу конфигурации).

---

## Рекомендуемая архитектура (без затрат)

### На телефоне (Android, офлайн)
1. **Камера**: CameraX (Kotlin).
2. **Детекция области ценника**:
   - этап 1 (быстрый старт): геометрия/контуры + стабилизация кадра;
   - этап 2 (точнее): лёгкая модель детекции (YOLOv8n / YOLO11n в TFLite/ONNX).
3. **OCR**:
   - базовый вариант: ML Kit Text Recognition (on-device, бесплатно);
   - альтернатива: PaddleOCR/Tesseract (сложнее интеграция, иногда выше контроль).
4. **Парсинг полей**:
   - постобработка регулярками + эвристики;
   - проверка диапазонов (например, цена < 100000).
5. **Редактирование результата**:
   - экран подтверждения/правки перед сохранением.
6. **Хранение**:
   - локально Room(SQLite), с сущностью `session` и сущностью `scan_item`.
7. **Экспорт**:
   - CSV (обязательно для MVP);
   - Google Sheets — второй этап (через API после MVP).

### На ПК (Python)
- Подготовка/очистка датасета (если понадобится детекция моделью).
- Скрипты оценки точности OCR + парсинга на реальных фото.
- (Опционально) дообучение детектора ценника.

---

## Нужно ли обучение нейросети?

Короткий ответ: **для первого MVP чаще всего нет**.

### Что можно сделать без обучения
1. Находим ценник в кадре по прямоугольной области + автофокус/стабильность.
2. Запускаем OCR на этой области.
3. Из OCR-текста вытаскиваем поля правилами.
4. Даём экран ручной правки.

### Когда обучение реально нужно
- много разных шаблонов ценников;
- сложные ракурсы/блики/фон;
- требование к уверенной live-детекции без ручного подтверждения.

Тогда обучаем **только детектор ценника**, а OCR оставляем готовый.

## Варианты OCR: что выбрать

### 1) Google ML Kit (on-device)
**Плюсы**:
- работает на устройстве;
- бесплатный;
- простая интеграция в Android;
- хорошая скорость для live-режима.

**Минусы**:
- ограниченный контроль и нет классического дообучения под кейс.

### 2) Tesseract
**Плюсы**:
- полностью open-source;
- гибкая настройка препроцессинга.

**Минусы**:
- часто хуже качество на сложных мобильных фото без серьёзной обработки;
- сложнее получить стабильный live UX.

### 3) PaddleOCR
**Плюсы**:
- обычно очень сильное качество OCR.

**Минусы**:
- интеграция на Android сложнее;
- могут быть выше требования к ресурсам.

**Рекомендация для старта**: ML Kit on-device + хорошая предобработка и строгий постпарсинг.

## План разработки и декомпозиция задач

Подробный backlog с параллельными треками: **[`docs/TASKS.md`](docs/TASKS.md)**.

Коротко по этапам:
1. Этап 0 — подготовка датасета/правил парсинга.
2. Этап 1 — базовый OCR pipeline и ручная правка.
3. Этап 2 — live-поток и стабилизация.
4. Этап 3 — сессии + мастер-таблица + CSV экспорт.
5. Этап 4 — улучшение точности (опциональное дообучение детектора).

---

## Критерии успеха MVP

- не менее 85–90% корректного извлечения полей на твоих шаблонах;
- ручная правка доступна всегда;
- live-режим работает хотя бы 1 распознавание в 1–3 сек;
- установка APK на Android 13 без облака.
